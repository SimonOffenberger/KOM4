\section{Fragen}
\subsection{Wofür wird FAULT QUEUE verwendet?}
Die \textit{Fault Queue} dient als Filter, um Fehlauslösungen des \texttt{ALERT}-Pins zu reduzieren. Dazu muss die gemessene Temperatur \(T_A\) für eine einstellbare Anzahl aufeinanderfolgender Wandlungszyklen oberhalb von \(T_{SET}\) liegen, bevor \texttt{ALERT} aktiviert wird. Entsprechend muss \(T_A\) auch für die eingestellte Anzahl von Zyklen unterhalb von \(T_{HYST}\) liegen, bevor \texttt{ALERT} wieder deaktiviert wird bzw. ein neuer Interrupt ausgelöst werden kann.

\subsection{Wie viele Temperatursensoren können parallel maximal am I2C-Bus betrieben werden?}
Am I\textsuperscript{2}C-Bus können maximal \textbf{acht} MCP9800/1/2/3-Sensoren parallel betrieben werden, da es \textbf{3} adressierbare Pins (A0, A1, A2) gibt, die jeweils mit GND oder VDD verbunden werden können. Dadurch ergeben sich \(2^3 = 8\) mögliche Adresskombinationen für die Sensoren.

\subsection{Welche Auflösung des Sensors ist nach dem Reset eingestellt?}
Nach dem Reset bzw. nach Power-up ist standardmäßig eine Auflösung von \textbf{9 Bit} eingestellt. Das entspricht einer Temperaturauflösung von \textbf{0{,}5\,$^\circ$C}.
\\\\

\subsection{Wozu kann der One-Shot Mode verwendet werden?}
Der \textit{One-Shot Mode} wird für \textbf{stromsparende Anwendungen} verwendet. In diesem Modus führt der Sensor auf Anforderung genau \textbf{eine einzelne Temperaturmessung} durch und kehrt danach automatisch wieder in den \textit{Shutdown Mode} zurück. Das ist besonders nützlich, wenn die Temperatur nur gelegentlich und gezielt durch einen Controller abgefragt werden soll.

\section{Funktionsbeschreibung}
Im Projekt wird die aktuelle Temperatur über I2C vom MCP9800 Sensor ausgelesen und auf einem LCD-Display angezeigt. Zusätzlich wird ein Alarm ausgelöst, wenn die gemessene Temperatur den eingestellten Schwellenwert überschreitet. Der \texttt{ALERT}-Pin des Sensors ist mit einem externen Interrupt verbunden, der bei Aktivierung eine LED einschaltet, um den Alarm visuell anzuzeigen. Die Temperaturwerte werden kontinuierlich aktualisiert.
Bei aktivem Alarm (Temperatur über \(T_{SET}\)) leuchtet die LED4 auf, und bei Rückkehr unter \(T_{HYST}\) erlischt sie wieder. 
\subsection{MCP Driver}
Der Treiber für den MCP9800 Temperatursensor baut auf den I2C Treiber auf.
Dieser ist so ausgelegt, dass mehrere Sensoren parallel am I2C-Bus betrieben werden können.
Dazu muss für jeden Sensor ein eigener Handle angelegt werden.
Im Handle werden im Bereich \textbf{init} der Initialisierungszustand des MCP9800 gesetzt und anschließend mit der Funktion \texttt{MCP9800\_Temp\_Init} initialisiert.
Bei jedem Aufruf einer Treiber Funktion ist der entsprechende Handle zu übergeben.
Status der Funktion wird über den Rückgabewert der Funktion zurückgegeben.
Hier wird die I2C\_RetType verwendet, um Fehler zu erkennen und entsprechend zu reagieren.
